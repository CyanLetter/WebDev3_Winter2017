Transitions

Simple animation from point A to B

Properties


transition: width 1s ease 1s;

/* is shorthand for */
transition-property: width;
transition-duration: 1s;
transition-timing-function: ease;
transition-delay: 1s;

Triggers

Any change to an element's properties.

Pseudo-classes such as :hover and :target.

Styles applied through media queries.

transition-property

Animatable properties only.

Generally only between absolute values.

Once again, these can be in a comma separated list.



#div1 {
	transition-property: width, height, transform;	
}

transition-duration

Duration of the transition.

Defaults to 0 - always specify this.

transition-timing-function

Giving life to transitions



#div1 {transition-timing-function: linear;}
#div2 {transition-timing-function: ease;}
#div3 {transition-timing-function: ease-in;}
#div4 {transition-timing-function: ease-out;}
#div5 {transition-timing-function: ease-in-out;}
#div6 {transition-timing-function: cubic-bezier(.19, .84, .82, .41);}

cubic-bezier.com/

transition-delay

Delay of the transition.

Can be used for simple sequencing.

Practical Applications

Google Inbox

  • Fade In Up Sequence
  • Flip or Expand and Reveal

The Properties


#div1 {
	background-color: transparent;
	background-image: url('path/to/image.png');
	background-repeat: repeat;
	background-position: 0% 0%;
	background-attachment: scroll;
	background-size: auto;
}

background-image

Used for large images, patterns, and gradients

background-repeat

repeat, no-repeat, repeat-x, repeat-y

Neat effects!

background-clip

Expand clickable area


.close-button {
	width: 400px;
	height: 400px;
	border-radius: 200px;
	background: white;
	padding: 170px; /* 60px visible area */
	background-clip: content-box;
}

background-position


#div1 {
	background-image: url('path/to/image.png');
	background-position: center top;
}
#div2 {
	background-image: url('path/to/image.png');
	background-position: 10% 20%;
}
#div3 {
	background-image: url('path/to/image.png');
	background-position: 30px 80px;
}

defaults to 0% 0%

background-size

length, percentage, cover, contain

Undeclared values are set to auto

Be careful to maintain your proportions


#div1 {
	background-size: 200px 400px;
}
#div2 {
	background-size: 100%;
}
#div3 {
	background-size: cover;
}

background-attachment

scroll, local, fixed


#div1 {
	background-attachment: scroll;
	/* Default, moves relative to entire document */
}
#div2 {
	background-attachment: local;
	/* Relative to its container */
}
#div3 {
	background-attachment: fixed;
	/* Relative to viewport, like fixed positioning */
}

Layering

Can comma separate multiple backgrounds

Use for textures or effect layering



#div1 {
	background-image: 
		url('top-image.png'),
		url('next-image.png');
	background-repeat: repeat, no-repeat;
	background-size: auto, cover;
}

#div2 {
	background-image:
		linear-gradient(transparent, white),
		url('image.png');
		background-repeat: no-repeat;
}
Patternify for Backgrounds

Intelligent Loading

Only load the size you need


#div1 {
	/* Don't put in a background-image yet */
	background-repeat: no-repeat;
	background-position: center;
	background-size: cover;
}

@media screen and (max-width: 480px) {
	#div1 {
		background-image: url('smaller-image.png');
	}
}

@media screen and (min-width: 481px) {
	#div1 {
		background-image: url('bigger-image.png');
	}
}

:before and :after

Are very exciting!

Act like inline elements.

Require content property to generate.

Can't be used on 'replaced' elements.

Content

Values can be String, url(), attr(), counter(), open-quote, close-quote...

Special characters are weird. 



.classOne:before {
	content: "A String";
}
.classOne:before {
	/* Copyright sign character code */
	content: "\00A9";
}
.classTwo:before {
	content: url('myImage.jpg');
}
.classThree:before {
	content: attr(id);
}

Why should I care?

It keeps your markup clean.

It gives you re-usable modules.

It helps remove the need for images.

It helps maintain the separation of style and content.
